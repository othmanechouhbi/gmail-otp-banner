# Gmail OTP Banner

Gmail OTP Banner is an open-source Chrome extension that helps you copy recent verification codes from Gmail faster.

## Project Status

Gmail OTP Banner is currently shared as an open-source project for local installation.

It is not available on the Chrome Web Store yet. Users must download the source code, configure their own Google Cloud project, create their own OAuth Client ID, and install the extension manually using Chrome Developer Mode.

## What The Extension Does

- Connects to Gmail using Google OAuth.
- Detects new verification codes received after you connect an account.
- Displays the newest valid OTP in a small banner.
- Runs locally in the browser.
- Uses Gmail read-only access and never asks for your Gmail password.

## Requirements

- Google Chrome.
- Google account / Gmail account.
- Google Cloud project.
- Gmail API enabled.
- OAuth Client ID for Chrome Extension.

## Manual Installation Guide

1. Download or clone this repository.
2. Open `chrome://extensions/` in Google Chrome.
3. Enable **Developer Mode**.
4. Click **Load unpacked**.
5. Select the `extension` folder from this project.
6. Copy the generated Chrome Extension ID shown on the extension card.
7. Go to the Google Cloud Console.
8. Create a Google Cloud project.
9. Enable the Gmail API.
10. Configure the OAuth consent screen.
11. Add yourself as a test user.
12. Create an OAuth Client ID of type **Chrome Extension**.
13. Paste the Chrome Extension ID from Chrome.
14. Copy the generated OAuth Client ID.
15. Open `extension/manifest.json`.
16. Replace `YOUR_CHROME_EXTENSION_OAUTH_CLIENT_ID` with your OAuth Client ID.
17. Reload the extension from `chrome://extensions/`.
18. Click the Gmail OTP Banner icon.
19. Click **Connect an account**.
20. Sign in with Google and allow Gmail read-only access.

## Google Cloud Setup Guide

Use these paths in the Google Cloud Console:

- `Google Cloud Console → APIs & Services → Library → Gmail API → Enable`
- `Google Cloud Console → Google Auth Platform → Branding`
- `Google Cloud Console → Google Auth Platform → Audience → Test users`
- `Google Cloud Console → Google Auth Platform → Clients → Create Client → Chrome Extension`

When creating the OAuth Client ID, choose **Chrome Extension** as the application type and paste the Chrome Extension ID generated by Chrome after loading the unpacked extension.

## Important Note About Extension ID

The local Chrome Extension ID must match the Chrome Extension ID used in your Google OAuth Client configuration.

If you remove the extension and load it again, Chrome may generate a different Extension ID. If the ID changes, Google OAuth may fail with `redirect_uri_mismatch`.

To avoid this, keep the same extension loaded in Chrome and use **Reload** from `chrome://extensions/` after editing files instead of removing and loading the extension again.

## Common Errors And Fixes

| Error | Fix |
| --- | --- |
| `Error 403: access_denied` | Add your Gmail address as a test user in `Google Cloud Console → Google Auth Platform → Audience → Test users`. |
| `Error 400: redirect_uri_mismatch` | Make sure the local Chrome Extension ID exactly matches the ID saved in your OAuth Client. |
| `access_type offline not allowed` | Remove `access_type=offline`. Chrome extension OAuth should not request offline access for this setup. |
| `invalid client_id` | Check that `extension/manifest.json` contains the correct OAuth Client ID. |
| Gmail API not enabled | Enable Gmail API at `Google Cloud Console → APIs & Services → Library → Gmail API → Enable`. |

## Privacy

Gmail OTP Banner processes Gmail data locally in your browser.

The extension only scans recent Gmail messages for OTP detection. It does not sell data, does not send Gmail content to external servers, and does not store full email bodies.

The extension stores only small local data needed to work, such as connected account information, recent message IDs, ignored message IDs, and the latest detected OTP.
