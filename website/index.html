Gmail OTP Banner

Gmail OTP Banner is an open-source Chrome extension that connects to Gmail using Google OAuth, detects new verification codes received after connection, and displays the newest valid OTP in a small browser banner.

Current status: Gmail OTP Banner is not available on the Chrome Web Store yet. Clone the repository from GitHub and install the extension manually using Chrome Developer Mode.

What It Does

  • Connects to Gmail using Google OAuth.
  • Scans recent Gmail messages for OTP detection.
  • Shows the newest valid OTP in a small banner.
  • Runs locally in your browser.
  • Does not send Gmail content to external servers.

Manual Installation Guide

1. Clone the repository

Users need Git installed.

git clone https://github.com/YOUR_USERNAME/gmail-otp-banner.git
cd gmail-otp-banner

Replace YOUR_USERNAME with the real GitHub username. If you do not have Git, click Code -> Download ZIP on GitHub, extract the ZIP, and open the extracted folder.

2. Load the extension in Chrome Developer Mode

  1. Open Google Chrome.
  2. Go to chrome://extensions/.
  3. Enable Developer mode in the top-right corner.
  4. Click Load unpacked.
  5. Select the folder named extension.
  6. Copy the generated Extension ID.

Important: select the extension folder, not the full repository root folder. The Extension ID will look like abcdefghijklmnopabcdefghijklmnop.

3. Create or access Google Cloud Console

  1. Go to https://console.cloud.google.com/.
  2. Sign in with your Google/Gmail account.
  3. If you do not have a Google Cloud account yet, create one using your Gmail account.
  4. Accept the Google Cloud terms if Google asks you to.

4. Create a Google Cloud project

  1. Click the project selector at the top of Google Cloud Console.
  2. Click New Project.
  3. Use Gmail OTP Banner or OTP Extension as the project name.
  4. Choose No organization if you are using a personal account.
  5. Click Create and wait until the project is ready.
  6. Make sure the new project is selected.

5. Enable Gmail API

  1. Go to https://console.cloud.google.com/apis/dashboard.
  2. Make sure your new project is selected.
  3. Click Enable APIs and Services.
  4. Search for Gmail API.
  5. Open Gmail API and click Enable.

6. Configure Google Auth Platform

  1. Go to https://console.cloud.google.com/auth/overview.
  2. Click Get started.
  3. Set the app name to Gmail OTP Banner.
  4. Select your email as the user support email.
  5. Choose External as the audience.
  6. Enter your email for contact information.
  7. Accept the Google API Services User Data Policy checkbox.
  8. Click Create.

7. Add your Gmail account as a test user

  1. In Google Auth Platform, go to Audience.
  2. Find Test users.
  3. Click Add users.
  4. Enter your Gmail address.
  5. Click Save.

If you do not add your email as a test user, Google may show Error 403: access_denied.

8. Create OAuth Client ID

  1. In Google Auth Platform, go to Clients.
  2. Click Create Client.
  3. Choose Chrome Extension as the application type.
  4. Name it Gmail OTP Banner.
  5. Paste the Extension ID copied from chrome://extensions/.
  6. Click Create.
  7. Copy the generated Client ID.

The Client ID looks like xxxxxxxxxxxx-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx.apps.googleusercontent.com.

9. Add your OAuth Client ID to the extension

  1. Open extension/manifest.json.
  2. Find "client_id": "YOUR_CHROME_EXTENSION_OAUTH_CLIENT_ID".
  3. Replace it with your own Client ID.
  4. Save the file.

10. Reload the extension

  1. Go back to chrome://extensions/.
  2. Find Gmail OTP Banner.
  3. Click Reload.
  4. Open the extension popup.
  5. Click Connect an account.
  6. Sign in with the same Gmail account you added as a test user.
  7. Grant permission.

11. Test the extension

  1. Open any service that sends a verification code to your Gmail.
  2. Request a new verification code.
  3. The extension should show a small floating banner.
  4. Click Copy to copy the OTP.

The extension does not show old codes received before connection. It only shows new OTP codes received after connection, and the banner automatically disappears after a few seconds.

Troubleshooting

Error Fix
Error 400: redirect_uri_mismatch Create a new Chrome Extension OAuth Client using the exact Extension ID from chrome://extensions/, then paste the new Client ID into extension/manifest.json.
Error 403: access_denied Add your Gmail address under Google Auth Platform test users.
Gmail API not enabled Enable Gmail API in Google Cloud -> APIs & Services -> Library.
invalid client_id Copy the correct Client ID from Google Cloud, paste it into extension/manifest.json, save, and reload the extension.
access_type offline not allowed Remove access_type=offline from the OAuth request.
Extension ID changed Use Reload instead of Remove. If the ID changed, create a new OAuth Client ID with the new Extension ID.

Privacy note

  • Uses Google OAuth only.
  • Never asks for your Gmail password.
  • Reads recent Gmail messages only to detect OTP codes.
  • Processes OTP detection locally in the browser.
  • Does not send Gmail content to any external server.
  • Does not sell or share Gmail data.

Future Chrome Web Store Publication

Chrome Web Store publication may be added in the future, but currently users must install Gmail OTP Banner manually from the source code.
