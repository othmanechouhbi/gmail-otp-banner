Support

Gmail OTP Banner is currently shared as an open-source local installation project. It is not available on the Chrome Web Store yet.

Clone the repository from GitHub and install the extension manually using Chrome Developer Mode. Each user must create their own Google Cloud project and OAuth Client ID.

Quick Setup Checklist

  1. Clone the repository with Git, or download the ZIP from GitHub.
  2. Open chrome://extensions/.
  3. Enable Developer mode.
  4. Click Load unpacked and select the extension folder, not the full repository root folder.
  5. Copy the generated Extension ID.
  6. Go to https://console.cloud.google.com/.
  7. Create a Google Cloud project named Gmail OTP Banner or OTP Extension.
  8. Enable Gmail API.
  9. Configure Google Auth Platform with an External audience.
  10. Add your Gmail address as a test user.
  11. Create an OAuth Client ID of type Chrome Extension using the Extension ID from Chrome.
  12. Paste the generated Client ID into extension/manifest.json.
  13. Reload the extension.
  14. Click Connect an account and sign in with the test user Gmail account.

Git Clone Command

git clone https://github.com/YOUR_USERNAME/gmail-otp-banner.git
cd gmail-otp-banner

Replace YOUR_USERNAME with the real GitHub username. If you do not have Git, click Code -> Download ZIP on GitHub, extract the ZIP, and open the extracted folder.

Important Extension ID Note

The local Extension ID must match the OAuth Client Chrome Extension ID in Google Cloud. The ID will look like abcdefghijklmnopabcdefghijklmnop.

If you remove and load the extension again, the ID can change. If the ID changes, OAuth may fail with redirect_uri_mismatch. Keep the same extension loaded and use Reload instead of Remove.

Google Cloud Paths

  • Google Cloud Console -> APIs & Services -> Library -> Gmail API -> Enable
  • Google Cloud Console -> Google Auth Platform -> Branding
  • Google Cloud Console -> Google Auth Platform -> Audience -> Test users
  • Google Cloud Console -> Google Auth Platform -> Clients -> Create Client -> Chrome Extension

Troubleshooting

Error Cause Fix
Error 400: redirect_uri_mismatch The Extension ID in Google Cloud does not match the Extension ID in Chrome. Copy the current Extension ID from chrome://extensions/, create a new Chrome Extension OAuth Client with that exact ID, copy the new Client ID, paste it into extension/manifest.json, and reload the extension.
Error 403: access_denied Your Gmail account is not added as a test user. Go to Google Auth Platform Audience, add your Gmail address under Test users, save, and try again.
Gmail API not enabled The Gmail API is not enabled for the selected project. Go to Google Cloud APIs & Services Library, search Gmail API, and click Enable.
invalid client_id The Client ID in manifest.json is wrong or belongs to another project. Copy the correct Client ID from Google Cloud, paste it into extension/manifest.json, save, and reload the extension.
access_type offline not allowed Chrome extension OAuth should not request offline access for this setup. Remove access_type=offline from the OAuth request.
Extension ID changed The extension was removed and loaded again. Avoid removing the extension during setup. Use Reload instead. If the ID changed, create a new OAuth Client ID with the new Extension ID.

Testing The Extension

  1. Open any service that sends a verification code to your Gmail.
  2. Request a new verification code.
  3. The extension should show a small floating banner.
  4. Click Copy to copy the OTP.

The extension does not show old codes received before connection. It only shows new OTP codes received after connection, and the banner automatically disappears after a few seconds.

Privacy Note

  • Uses Google OAuth only.
  • Never asks for your Gmail password.
  • Reads recent Gmail messages only to detect OTP codes.
  • Processes OTP detection locally in the browser.
  • Does not send Gmail content to any external server.
  • Does not sell or share Gmail data.
