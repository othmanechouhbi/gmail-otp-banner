# Security Policy

## Supported Versions

Security updates are provided for the latest published version of Gmail OTP Banner.

## Reporting a Vulnerability

Please report security issues privately before opening a public issue. Include:

- A clear description of the vulnerability.
- Steps to reproduce.
- Impact assessment.
- Browser and extension version.
- Any relevant screenshots or logs, with sensitive information redacted.

Contact: otmanechouhbi@gmail.com

## Security Design

Gmail OTP Banner follows these principles:

- Uses Google OAuth instead of passwords.
- Requests only Gmail readonly access.
- Processes Gmail message content locally in the browser.
- Does not send Gmail content to external servers.
- Stores only minimal operational data.
- Avoids third-party analytics and advertising code.

## Vulnerability Handling

Valid reports will be acknowledged as soon as possible. Fixes will be prioritized based on severity and released through the Chrome Web Store after testing.
